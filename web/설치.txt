manage.py 파일이 있는 가장 바깥 폴더에 아래 파일을 새로 만드세요. (파일명: db_uploader.py)

Python
import os
import json
import django

# 1. Django 환경 설정 (내 프로젝트 이름을 가져옵니다)
os.environ.setdefault("DJANGO_SETTINGS_MODULE", "my_quiz_site.settings")
django.setup()

from quiz.models import Question, Choice

def insert_quiz_data(json_file_path):
    # 2. JSON 파일 읽기
    with open(json_file_path, 'r', encoding='utf-8') as f:
        data = json.load(f)

    print(f"{len(data)}개의 데이터를 DB에 넣기 시작합니다...")

    for item in data:
        # 3. Question(문제) 저장
        q = Question.objects.create(
            number=int(item['no']),
            content=item['content']
        )

        # 4. Choice(보기) 저장
        # 객관식 보기가 있는 경우에만 실행합니다.
        for opt in item.get('options', []):
            Choice.objects.create(
                question=q,
                choice_text=opt
            )
        
    print("모든 데이터가 성공적으로 저장되었습니다!")

if __name__ == "__main__":
    # 아까 생성된 JSON 파일 이름을 여기에 적으세요!
    target_json = "result_파일명.json" 
    insert_quiz_data(target_json)

3단계: settings.py 수정
이제 my_quiz_site/my_quiz_site/settings.py 파일을 열어 DATABASES 섹션을 SQLite에서 PostgreSQL로 교체합니다.

Python
# settings.py

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'quiz_db',          # 생성한 DB 이름
        'USER': 'postgres',         # 기본 사용자 이름 (또는 본인이 설정한 이름)
        'PASSWORD': 'your_password', # 설정한 비밀번호
        'HOST': '127.0.0.1',        # 로컬 환경
        'PORT': '5432',             # PostgreSQL 기본 포트
    }
}
4단계: 데이터 이관 (SQLite → PostgreSQL)
DB가 완전히 바뀌었으므로, 새로운 DB에 테이블 구조를 만들고 기존 데이터를 옮겨야 합니다.

새 DB에 테이블 만들기:

Bash
python manage.py migrate
데이터 옮기기 (선택 사항):
만약 기존 SQLite의 데이터를 그대로 가져오고 싶다면:

SQLite에서 추출: python manage.py dumpdata > data.json

PostgreSQL로 주입: python manage.py loaddata data.json

주의: 이 과정에서 데이터 타입이나 인코딩 문제로 에러가 날 수 있습니다. 에러가 나면 db_uploader.py를 다시 돌리는 것이 더 깔끔합니다.

정확한 수행 시점과 절차를 정리해 드릴게요.

📅 권장 수행 시점: "데이터 업로드 직후"
데이터 관리 흐름을 보면 다음과 같습니다.

시험지 ID 확인 (Admin 페이지)

관리자 페이지에서 방금 생성된 시험지의 ID(예: 5번)를 확인합니다.

이미지 파일 이름 변경

준비한 이미지들을 규칙(5_1.jpg, 5_1_1.jpg 등)에 맞춰 바꿉니다.

이미지 매칭 수행 (image_matcher.py) 👈 바로 이 시점!

DB에 이미 Question과 Choice 레코드가 존재해야 이미지를 연결할 수 있기 때문입니다.

일괄 편집 화면에서 검수 (bulk_edit.html)

이미지가 잘 들어갔는지 🖼️ 아이콘으로 최종 확인하고 정답을 보정합니다.

💡 운영 팁: 더 효율적인 관리를 위한 조언
지금 구축된 시스템을 더 스마트하게 활용하는 방법입니다.

시험지 ID 식별: bulk_edit 화면 상단에 시험지 ID를 크게 표시해 두면, 이미지 파일명을 바꿀 때 헷갈리지 않고 작업할 수 있습니다.

데이터 무결성: bulk_edit 화면에서 정답이 비어있는 행만 눈에 띄게 (예: 배경색 노란색) 처리해 두면, 검수 누락을 100% 방지할 수 있습니다.

이미지 폴더 관리: 매칭이 끝난 이미지 폴더는 completed/ 같은 하위 폴더로 옮겨 관리하면, 다음 회차 작업 시 이미지가 섞이는 것을 방지할 수 있습니다.


구글 클라우드 비전(Google Cloud Vision) API를 사용하기 위해서는 단순히 구글 계정만 있다고 되는 것은 아니고, **구글 클라우드 플랫폼(GCP)**이라는 기업용 클라우드 환경 내에서 몇 가지 설정을 거쳐야 합니다.

절차가 조금 복잡해 보일 수 있지만, 순서대로 따라오시면 금방 끝납니다.

1. 구글 클라우드 플랫폼(GCP) 가입 및 프로젝트 생성
GCP 콘솔 접속: Google Cloud Console에 접속합니다.

프로젝트 생성: 상단 메뉴에서 프로젝트 선택 -> 새 프로젝트를 클릭하여 기출문제 관리용 프로젝트(예: quiz-system-ocr)를 만듭니다. 모든 서비스는 이 '프로젝트' 단위로 묶입니다.

2. 결제 프로필(Billing) 등록 (필수)
무료 한도(월 1,000건)만 사용하더라도 결제 수단 등록은 필수입니다.

준비물: 해외 결제 가능한 신용카드/체크카드.

이유: 무분별한 API 호출(DDoS 등)을 방지하고 무료 한도 초과 시 과금을 하기 위한 구글의 정책입니다.

팁: 신규 가입 시 보통 $300의 무료 크레딧을 주므로 초기에는 전혀 비용 걱정을 안 하셔도 됩니다.

3. Vision API 활성화
프로젝트 안에서 "비전 API를 사용하겠다"고 선언해야 합니다.

콘솔 왼쪽 메뉴에서 **[API 및 서비스] > [라이브러리]**로 이동합니다.

검색창에 **"Cloud Vision API"**를 입력합니다.

해당 API를 선택하고 [사용해 보기(Enable)] 버튼을 클릭합니다.

4. 서비스 계정(Service Account) 및 키 발급
파이썬(Django) 서버가 구글 서버에 "나 허락받은 사용자야"라고 증명할 **'열쇠'**가 필요합니다.

[IAM 및 관리자] > [서비스 계정] 메뉴로 이동합니다.

**[서비스 계정 만들기]**를 클릭하여 이름을 정합니다.

역할(Role) 선택 시 [Cloud Vision AI] > [Cloud Vision AI 관리자] 권한을 부여합니다.

만들어진 계정을 클릭하고 **[키(Keys)] > [키 추가] > [새 키 만들기]**를 선택합니다.

JSON 방식으로 생성하면 파일이 하나 다운로드됩니다. (이 파일이 가장 중요합니다! 절대 외부에 유출하면 안 됩니다.)

5. 파이썬(Django) 환경 설정
이제 다운로드한 JSON 키 파일을 프로젝트 폴더(예: my_quiz_site/keys/)에 넣고, 라이브러리를 설치합니다.

Bash
# 구글 클라우드 비전 라이브러리 설치
pip install google-cloud-vision
그 후 views.py에서 이 키 파일을 참조하여 API를 호출하게 됩니다.

통합 운영 센터의 핵심 기능
지능형 분기 처리: 사용자가 '필기'를 선택하면 이미지 OCR 엔진이 작동하고, '실기'를 선택하면 PDF 텍스트 분석 엔진이 작동합니다.

원스톱 대시보드: 단순히 업로드만 하는 것이 아니라, 현재 등록된 문제 수와 정답 누락 건수 등을 한눈에 모니터링할 수 있습니다.

데이터 무결성: 실기 PDF 파싱 후 정답이 매칭되지 않은 데이터는 '정답 미비' 통계에 표시되어 관리자가 즉시 보완할 수 있게 돕습니다.

✨ 통합 운영 센터의 변화동적 엔진 전환: 업로드 시 '실기'를 선택하면 PyMuPDF가 작동하여 PDF 텍스트를 파싱하고, '필기'를 선택하면 기존의 Google Vision OCR 로직이 가동됩니다.데이터 일원화: 어떤 방식으로 문제를 등록하든 최종적으로는 아래의 일괄 편집 테이블에 데이터가 쌓이므로, 등록 후 즉시 오타 수정이나 이미지 매칭을 할 수 있습니다.실기 특화 파싱: 실기 시험지의 과목명(알고리즘 등)과 문항 번호(①~⑮)를 인식하여 에 있는 과목별 정답과 자동으로 매칭을 시도합니다.
